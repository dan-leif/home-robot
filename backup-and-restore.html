💾 Backup & Restore

Double-click Backup Robot on your desktop, then pick a number:

1
Backup — save how everything is right now, as a new restore point.
2
View backups — see the restore points you've saved, with the date of each.
3
Restore — pick a backup from the list and roll the robot back to it. (It shuts the robot down for you, rolls back, then tells you to start it again.)

Tip: make a backup before any big change. If something breaks, Restore puts you back in a minute.

Your code is saved separately on GitHub automatically — this menu takes care of everything else.

Home Assistant setup
your pipeline, the "no follow-ups" prompt, integrations, add-ons		 A snapshot of the VM, made by Backup (option 1). This is the main thing the menu saves. Restore (option 3) rolls it back. It's the only copy of this, so back up before big changes.
Scripts & web pages
Start/Stop/Backup, these help pages, the voice server code		 Saved to GitHub automatically whenever changes are pushed. Already kept safe online — nothing for you to do.
The AI brain
the llama3.1 model		 Not saved (it's large) — but it's free to re-download. Re-downloads on its own with one command (ollama pull).
The voice
Kokoro "Sky" voice files		 Not saved (large) — free to re-download from the web. Re-download the two voice files into the kokoro folder.
Settings
env vars, firewall rules		 Written into a small note saved with every backup. Re-enter the handful of values from that note (or just re-run setup).

In short: Backup/Restore here handle the Home Assistant setup; everything else is either on GitHub or free to re-download.

Worried about losing the whole laptop? See Recovery & Off-site Backup to set up an off-laptop copy and to rebuild on a new machine.
