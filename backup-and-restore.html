💾 Backing up & restoring the robot

Every moving part, where it's saved, and how to get it back. (2026-06-25)

The robot is made of two kinds of things:

Code (scripts, web pages, the Kokoro server) is backed up by git + GitHub. State (the whole Home Assistant setup inside the VM) is backed up by a VirtualBox snapshot. Everything else is either documented or just re-downloadable. One script — backup-restore.ps1 — handles the snapshot and writes down the rest.

The one script: backup-restore.ps1

It does three things. The easiest backup is just double-clicking “Backup Robot” on the desktop.

Back up — safe to run anytime, even with everything on 
powershell -ExecutionPolicy Bypass -File backup-restore.ps1 backup
Takes a VM snapshot and saves a manifest into backups\. (Or double-click Backup Robot.)
See what you've got 
powershell -ExecutionPolicy Bypass -File backup-restore.ps1 list
Lists every snapshot and every saved manifest.
Restore — the only step that throws away the current VM 
powershell -ExecutionPolicy Bypass -File backup-restore.ps1 restore
powershell -ExecutionPolicy Bypass -File backup-restore.ps1 restore robot-20260625-143000
Rolls the VM back to the latest snapshot (or a named one). It refuses while the VM is on — click Stop Robot first, then run restore, then Start Robot. With no name it uses the most recent snapshot.

A good habit

Run a backup before any big change — before installing an add-on, switching the model, or editing HA config. If it breaks, restore puts you back in a minute. Snapshots are cheap (they only store what changed), so taking them often costs almost nothing.

Everything stays free & local. Code lives in git/GitHub; the VM lives in VirtualBox snapshots; the big model/voice files re-download for free. Nothing here depends on a paid service.
