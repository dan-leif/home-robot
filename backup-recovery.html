🛟 Recovery & Off-site Backup

For the worst case: the laptop is lost, stolen, or dies. (2026-06-25)

The everyday Backup Robot menu protects you against mistakes — roll back in place. But its snapshots live on this laptop, so they can't save you if the whole laptop is gone. For that you need a copy of your Home Assistant setup off the laptop. Set that up once (below), and Backup Robot will keep it current automatically.

Part 1 — Set up off-site backup (do this once)

In the Home Assistant browser
  1. Create a long-lived token. Click your name (bottom-left) → Security tab → scroll to Long-lived access tokens → Create Token → name it Backup Robot → copy the long string it shows (you only see it once). That's all HA needs — no add-ons to install.
In the Backup Robot menu
  1. Double-click Backup Robot → choose 4 (Off-site).
  2. Enter: your HA IP, the off-site folder (press Enter to accept the default D:\My Drive\Dev\Robot\Backups — your Google Drive, so it leaves the laptop), then paste the token.
  3. Press 1 (Backup) once, then check your Google Drive folder — a .tar file (a few hundred MB) should appear.

From then on, every Backup asks Home Assistant to make a fresh full backup and downloads it into that Google Drive folder (keeping the newest 5). Nothing to remember — it just happens.

Part 2 — Rebuild on a new laptop

  1. Install the tools: VirtualBox, Ollama, and Python 3.12.
  2. Get the code: git clone https://github.com/dan-leif/home-robot into C:\DEV\home-robot. (All the scripts and these pages come back here.)
  3. Create a fresh Home Assistant VM in VirtualBox (download the HA OS image; bridged network, EFI on). The exact VBoxManage steps are in the project notes.
  4. Restore your setup: open the new HA, do the short onboarding, then Settings → System → Backups → Upload backup → pick the .tar from your off-site folder → Restore. Your pipeline, the "no follow-ups" prompt, integrations and add-ons all come back.
  5. Bring back the brain: ollama pull llama3.1:8b; set OLLAMA_HOST=0.0.0.0 and OLLAMA_KEEP_ALIVE=-1; allow firewall port 11434.
  6. Bring back the voice: re-download the two Kokoro files into kokoro\; allow firewall port 10200.
  7. Fix the address if the new PC's IP changed: update the HA Ollama integration to point at the new host IP.
  8. Double-click Start Robot. Done.

The one thing that must be off-site is your HA backup .tar (Part 1). Everything else — code, the AI model, the voice files — is on GitHub or free to re-download. Keep this page; it lives on GitHub, so it survives too.

Everything stays free & local. The off-site copy uses a folder you already sync (Google Drive) — no paid service, no extra add-ons.
