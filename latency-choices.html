🤖 Making the robot feel real-time

Where tonight's ~10-second delay actually comes from — and the plan to fix it. (2026-06-23)

What's slow (measured)

Warm — back-to-back
~1.6s
model already in memory
Cold — after a pause
~8s
model reloads into the GPU
The model generates fast (~47 words-worth/sec). The delay is two things: the model gets kicked out of GPU memory when idle and takes ~6.4s to reload, and the voice waits for the whole answer before it starts speaking.

The fix — three levers

  1. Pin the model in memory — so it never pays the 6.4s reload. Biggest single win.
    ✓ Claude does it · sets OLLAMA_KEEP_ALIVE=-1, no admin
  2. Stream the speech — start talking after the first sentence instead of waiting for the whole answer.
    → You do it · a setting in the Home Assistant UI
  3. Shorter answers — less to generate and less to read aloud.
    → You do it · tighten the Robot system prompt ("one or two short sentences")
